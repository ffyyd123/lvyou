你现在是一个资深分布式系统架构师 + Java后端工程师 + AI Agent工程师 + 前端工程师，需要从0到1设计并实现一个完整的“AI驱动旅游路线规划系统（MVP到可扩展架构）”。
本项目的目标不是普通旅游网站，而是一个：
👉 “AI驱动的旅行规划引擎 + 地图可视化执行系统”
系统核心能力是：用户输入旅行需求 → AI Agent生成结构化路线 → 前端地图可视化展示。
--------------------------------------------
# 🧠 一、项目目标（必须严格遵守）
系统必须实现以下能力：
1. 用户输入旅行需求：
    - 出发城市
    - 目的城市
    - 天数
    - 偏好（历史/自然/美食等）
2. AI生成“可执行旅行路线”：
    - 按天规划（Day1 - DayN）
    - 每天包含多个景点（POI）
    - 每个POI必须包含经纬度
    - 自动计算行程距离与时间
3. 前端地图可视化：
    - 使用高德地图展示路线
    - marker标记景点
    - polyline连接路线
    - 按天分颜色展示
4. AI必须具备推理能力：
    - 支持ReAct机制（思考 → 行动 → 观察 → 输出）
    - 支持工具调用（POI查询 / 距离计算 / 路线校验）
    - 支持记忆（用户偏好 + 当前会话）
--------------------------------------------
# 🧠 二、系统总体架构（强制要求）
系统分为三层：
--------------------------------------------
## 1️⃣ Java业务系统（Spring Boot）
职责：

- 接收用户请求

## 3️⃣ 前端系统（Vue 3）

- 输入旅行需求

--------------------------------------------
# 🧠 三、AI Agent设计（核心重点）
## 1️⃣ Agent模式（必须实现）
采用ReAct模式：
Thought → Action → Observation → Final Answer
AI必须通过多步推理生成路线，而不是一次性输出。
--------------------------------------------
## 2️⃣ Tools（工具系统）
必须实现以下工具：
### Tool 1：search_poi
- 输入：城市 + 关键词
- 输出：POI列表（名称 + 经纬度）
### Tool 2：calc_distance
- 输入：两个经纬度
- 输出：距离（km）
### Tool 3：validate_route
- 输入：一天的路线
- 输出：是否合理（时间/距离）
--------------------------------------------
## 3️⃣ Memory机制（必须）
使用轻量内存实现：
保存内容：
- 用户偏好（旅行类型、节奏）
- 当前会话路线
- 历史修改记录
  实现方式：
- Java Map 或 AgentScope Memory模块
--------------------------------------------
